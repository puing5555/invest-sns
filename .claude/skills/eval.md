# Eval 프로세스 (프롬프트 정확도 평가)

## 개요
인플루언서 시그널 분석 프롬프트의 정확도를 측정하고 버전 간 비교하는 체계.
69건 정답지 기반, Claude Sonnet으로 분류 후 정답 대비 비교.

## 정답지 (Ground Truth)

**파일**: `data/eval_ground_truth_69.json`
```json
[
  {
    "stock": "종목명",
    "key_quote": "발언 원문",
    "video_title": "영상 제목",
    "v12_result": "V12.3 결과 (기록용)",
    "correct": "정답 (수동 라벨링)",
    "reason_cat": "라벨링 사유 카테고리"
  }
]
```

### reason_cat 카테고리
| 카테고리 | 의미 |
|----------|------|
| 맥락일치 | 정답과 일치하는 판정 |
| 단순사례인용 | 다른 주제의 보조 예시로만 언급 → 중립 |
| 단순기술분석 | 기술 현황 설명일 뿐 투자 의견 없음 → 중립 |
| 리스크누락 | 리스크 언급을 무시하고 긍정으로 판정 |
| 조건부누락 | "~수도 있다" 가능성 표현을 확정으로 판정 |
| 시그널하향(제안수준) | 매수→긍정: "~하면 좋겠다"는 제안이지 강한 권유 아님 |
| 시그널하향(리스크언급) | 부정→중립: 리스크 분석은 확정 하락 선언 아님 |
| 시그널하향(단순질문) | 진행자 질문은 투자 의견 아님 → 중립 |
| 시그널상향 | 긍정→매수: 실제로는 더 강한 시그널 |
| 뷰유지(스탠스동일) | 이전 스탠스 유지 발언 |
| 화자불일치 | 게스트 발언을 채널 운영자로 잘못 귀속 |
| 종목불일치 | 다른 종목에 대한 발언을 잘못 매핑 |

### 3그룹 분석법 (프롬프트 개선 시 필수)

Eval 결과를 3그룹으로 분류하여 다음 버전 설계:

| 그룹 | 정의 | 활용 |
|------|------|------|
| A. 신규 정답 | 이전 오답 → 이번 정답 | **유지**: 이 규칙이 효과 있음 |
| B. 퇴보 | 이전 정답 → 이번 오답 | **수정**: 이 규칙이 부작용 발생 |
| C. 양쪽 오답 | 둘 다 오답 | **추가**: 새 규칙/few-shot 필요 |

### 프롬프트 개선 절차
1. 현재 버전 eval 실행 → 3그룹 추출
2. 각 그룹에서 대표 케이스 4~5개 선정
3. **A그룹**: 효과 있는 규칙 1~2줄로 압축하여 유지
4. **B그룹**: 과도한 규칙 완화 or 예외 조건 추가
5. **C그룹**: few-shot 예시로 커버
6. 새 프롬프트 조립 → eval 재실행 → 비교

## few-shot 예시 구성법

### 그룹별 배분
- 중립 판별 예시: 4~5개 (A그룹 + C그룹에서)
- 긍정 판별 예시: 3~4개 (B그룹에서)
- 매수 판별 예시: 2~3개 (B그룹 + C그룹에서)

### 예시 포맷
```
Q: 종목=종목명 / 영상="영상 제목"
발언: "핵심 발언"
A: {"signal_type": "판정"} ← 판정 근거 1줄
```

### 예시 선정 기준
- 가장 흔한 오분류 패턴 우선
- 경계가 모호한 케이스 (명확한 건 규칙으로 충분)
- 영상 제목이 판별에 중요한 케이스 (맥락 판별용)
