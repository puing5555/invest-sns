## 시그널 5단계 요약

| 타입 | 기준 | 핵심 표현 |
|------|------|-----------|
| **매수** | 강한 확신 + 투자 권유 | "사라", "추천합니다", "지금 들어가야" |
| **긍정** | 호의적이나 매수 권유 없음 | "좋은 종목", "전망 밝다", "담아보시면" |
| **중립** | 방향성 없음 / 보조 예시 | 단순 소개, 리스크 분석, 진행자 질문 |
| **부정** | 확정 하락 선언만 | "빠질 것이다", "투자하지 마세요" |
| **매도** | 직접 매도 권유 | "팔아라", "손절해라" |

## V14.0 핵심 판별 규칙 (3개)

### 1. 영상 맥락 판별 (가장 중요)
- 영상이 해당 종목을 **직접 분석** → 호의적 서술 = **긍정**
- 다른 주제 영상에서 **보조 예시**로만 언급 → **중립**
- 핵심 질문: "영상이 이 종목에 투자 관점의 분석을 하는가?"

### 2. 리스크 분석 ≠ 부정
- 리스크 언급, 우려, 가능성 제기 = **중립**
- "확정적 하락 선언 + 투자 회피 권고"만 = **부정**

### 3. 매수 vs 긍정 경계
- "~하면 좋겠다", "소개해 드리고 싶습니다" = **긍정** (제안)
- "~하세요", "추천합니다", 본인 매매 공개 = **매수** (권유)

## 프롬프트 수정 절차

### ⚠️ 수정 전 필수
1. 현재 eval 결과 확인 (`data/eval_v14_results.json`)
2. 수정 근거 3그룹 분석 완료 (→ `eval.md` 참조)
3. JAY 승인 (시그널 재분석 수반 시)

### 수정 프로세스
```
1. prompts/pipeline_vXX.md 새 파일 생성 (기존 파일 수정 X)
2. eval 스크립트에 새 프롬프트 반영
3. 69건 eval 실행 → 정확도 비교
4. regression 분석 (퇴보 건수 확인)
5. 승인 후:
   - pipeline_config.py 버전/경로 업데이트
   - CLAUDE.md 정확도 업데이트
   - 커밋
```

### 수정 시 주의사항
- **규칙 추가 시**: 반드시 few-shot 예시도 추가 (규칙만으로는 부족)
- **중립 필터 강화 시**: B그룹(퇴보) 확인 — 과도한 중립화 방지
- **매수 기준 변경 시**: 비트코인 케이스 특히 주의 (경계 모호)
- **few-shot 예시 수**: 10~15개 적정 (너무 많으면 토큰 낭비)

## 프롬프트 구조 (V14.0 기준)

```
1. 역할 정의
2. 시그널 타입 5단계 정의 (매수/긍정/중립/부정/매도)
3. 시그널 판별 핵심 규칙
   - 영상 맥락 판별
   - 시그널 제외 조건
   - 매수 vs 긍정 판별
4. 판별 예시 (few-shot 11개)
5. 1영상 1종목 1시그널 규칙
6. 종목 필터링 / 정규화
7. 화자 구분
8. 타임스탬프 / 필수 필드 규칙
9. 출력 형식 (JSON)
10. 변경 이력
```
